title: AuthX

The `AuthX` class is the core component of the AuthX library, providing comprehensive JWT management for FastAPI applications with support for access tokens, refresh tokens, cookie-based authentication, and CSRF protection.

## Constructor

### AuthX()

Initialize a new AuthX instance.

<ParamField path="config" type="AuthXConfig" default="AuthXConfig()">
  Configuration instance for AuthX behavior. If not provided, uses default configuration.
</ParamField>

<ParamField path="model" type="Optional[T]" default="None">
  Model type hint for type safety. Defaults to `dict[str, Any]` if not provided.
</ParamField>

```python
from authx import AuthX, AuthXConfig

# Basic initialization
auth = AuthX()

# With custom configuration
config = AuthXConfig(
    JWT_SECRET_KEY="your-secret-key",
    JWT_ACCESS_TOKEN_EXPIRES=timedelta(minutes=30)
)
auth = AuthX(config=config)

# With type hint for user model
from models import User
auth = AuthX[User](config=config)
```

## Properties

### config

Access the current AuthX configuration.

**Returns:** `AuthXConfig` - The configuration object

```python
config = auth.config
print(config.JWT_ALGORITHM)
```

### DEPENDENCY / BUNDLE

FastAPI dependency that provides an `AuthXDependency` object with request/response context.

**Returns:** `AuthXDependency[Any]` - Dependency for route injection

```python
from fastapi import Depends

@app.post("/login")
def login(bundle: auth.DEPENDENCY):
    token = bundle.create_access_token(uid="user123")
    bundle.set_access_cookies(token)
    return {"message": "Logged in"}
```

### ACCESS_REQUIRED

FastAPI dependency that enforces the presence of a valid access token.

**Returns:** `TokenPayload` - Decoded and verified token payload

**Raises:** `MissingTokenError`, `JWTDecodeError`

```python
@app.get("/protected")
def protected_route(payload: TokenPayload = Depends(auth.ACCESS_REQUIRED)):
    return {"user_id": payload.sub}
```

### REFRESH_REQUIRED

FastAPI dependency that enforces the presence of a valid refresh token.

**Returns:** `TokenPayload` - Decoded and verified token payload

```python
@app.post("/refresh")
def refresh_route(payload: TokenPayload = Depends(auth.REFRESH_REQUIRED)):
    new_token = auth.create_access_token(uid=payload.sub)
    return {"access_token": new_token}
```

### FRESH_REQUIRED

FastAPI dependency that enforces the presence of a fresh access token.

**Returns:** `TokenPayload` - Decoded and verified fresh token payload

```python
@app.delete("/account")
def delete_account(payload: TokenPayload = Depends(auth.FRESH_REQUIRED)):
    # Only fresh tokens can access this
    return {"deleted": payload.sub}
```

### ACCESS_TOKEN

FastAPI dependency that retrieves the access token from the request (optional).

**Returns:** `Optional[RequestToken]` - Token if present, None otherwise

```python
@app.get("/optional")
def optional_auth(token: Optional[RequestToken] = Depends(auth.ACCESS_TOKEN)):
    if token:
        return {"authenticated": True}
    return {"authenticated": False}
```

### REFRESH_TOKEN

FastAPI dependency that retrieves the refresh token from the request (optional).

**Returns:** `Optional[RequestToken]` - Token if present, None otherwise

### CURRENT_SUBJECT

FastAPI dependency that retrieves the currently authenticated subject/user.

**Returns:** `T` - The current subject (type depends on your model)

```python
@app.get("/me")
async def get_current_user(user: User = Depends(auth.CURRENT_SUBJECT)):
    return user
```

## Token creation

### create_access_token()

Generate a new access token.

<ParamField path="uid" type="str" required>
  Unique identifier for the user/subject
</ParamField>

<ParamField path="fresh" type="bool" default="False">
  Mark the token as fresh (for sensitive operations)
</ParamField>

<ParamField path="headers" type="Optional[dict[str, Any]]" default="None">
  Custom JWT headers
</ParamField>

<ParamField path="expiry" type="Optional[DateTimeExpression]" default="None">
  Custom expiry time (datetime or timedelta). Uses `JWT_ACCESS_TOKEN_EXPIRES` if not provided.
</ParamField>

<ParamField path="data" type="Optional[dict[str, Any]]" default="None">
  Additional data to embed in the token payload
</ParamField>

<ParamField path="audience" type="Optional[StringOrSequence]" default="None">
  Token audience claim
</ParamField>

<ParamField path="scopes" type="Optional[list[str]]" default="None">
  List of permission scopes to include in the token
</ParamField>

<ResponseField name="return" type="str">
  The encoded JWT access token
</ResponseField>

```python
# Basic usage
token = auth.create_access_token(uid="user123")

# Fresh token for sensitive operations
fresh_token = auth.create_access_token(uid="user123", fresh=True)

# Token with custom expiry
token = auth.create_access_token(
    uid="user123",
    expiry=timedelta(hours=2)
)

# Token with additional data
token = auth.create_access_token(
    uid="user123",
    data={"role": "admin", "department": "IT"}
)

# Token with scopes
token = auth.create_access_token(
    uid="user123",
    scopes=["users:read", "posts:write"]
)
```

## Cookie management

### set_access_cookies()

Add 'Set-Cookie' headers for the access token in the response.

<ParamField path="token" type="str" required>
  The access token to store in cookies
</ParamField>

<ParamField path="response" type="Response" required>
  FastAPI Response object to set cookies on
</ParamField>

<ParamField path="max_age" type="Optional[int]" default="None">
  Maximum age in seconds. Uses `JWT_COOKIE_MAX_AGE` if not provided.
</ParamField>

```python
from fastapi import Response

@app.post("/login")
def login(response: Response):
    token = auth.create_access_token(uid="user123")
    auth.set_access_cookies(token, response)
    return {"message": "Logged in"}
```

<Note>
  If `JWT_COOKIE_CSRF_PROTECT` is enabled, this also sets a CSRF token cookie.
</Note>

</ParamField>

<ParamField path="max_age" type="Optional[int]" default="None">
  Maximum age in seconds
</ParamField>

## Token retrieval

### get_token_from_request()

Retrieve a token from the incoming request.

<ParamField path="request" type="Request" required>
  FastAPI Request object
</ParamField>

<ParamField path="type" type="TokenType" default="'access'">
  Type of token to retrieve: `"access"` or `"refresh"`
</ParamField>

<ParamField path="optional" type="bool" default="True">
  If True, returns None when token is missing. If False, raises `MissingTokenError`.
</ParamField>

<ParamField path="locations" type="Optional[TokenLocations]" default="None">
  Specific locations to search for the token. Uses `JWT_TOKEN_LOCATION` if not provided.
</ParamField>

<ResponseField name="return" type="Optional[RequestToken]">
  The RequestToken if found, None if optional=True and not found
</ResponseField>

```python
# Optional access token
token = await auth.get_token_from_request(request)
if token:
    payload = auth.verify_token(token)

# Required refresh token
try:
    token = await auth.get_token_from_request(
        request,
        type="refresh",
        optional=False
    )
except MissingTokenError:
    # Handle missing token
    pass

# Search only in headers
token = await auth.get_token_from_request(
    request,
    locations=["headers"]
)
```

## Dependencies

### token_required()

Create a custom dependency to enforce token validation.

<ParamField path="type" type="str" default="'access'">
  Token type to require: `"access"` or `"refresh"`
</ParamField>

<ParamField path="verify_type" type="bool" default="True">
  Verify the token type
</ParamField>

<ParamField path="verify_fresh" type="bool" default="False">
  Require a fresh token
</ParamField>

<ParamField path="verify_csrf" type="Optional[bool]" default="None">
  Enable CSRF verification. Uses config default if None.
</ParamField>

<ParamField path="locations" type="Optional[TokenLocations]" default="None">
  Token locations to search
</ParamField>

<ResponseField name="return" type="Callable[[Request], Awaitable[TokenPayload]]">
  A FastAPI dependency function
</ResponseField>

```python
# Custom dependency requiring fresh token
fresh_required = auth.token_required(verify_fresh=True)

@app.delete("/account")
def delete_account(payload: TokenPayload = Depends(fresh_required)):
    return {"deleted": payload.sub}

# Only search in headers
header_auth = auth.token_required(locations=["headers"])

@app.get("/api/data")
def get_data(payload: TokenPayload = Depends(header_auth)):
    return {"data": "secure"}
```

### scopes_required()

Create a dependency that enforces specific permission scopes.

<ParamField path="*scopes" type="str" required>
  Variable number of required scope strings
</ParamField>

<ParamField path="all_required" type="bool" default="True">
  If True, ALL scopes must be present (AND logic). If False, at least ONE scope must be present (OR logic).
</ParamField>

<ParamField path="verify_type" type="bool" default="True">
  Verify token type
</ParamField>

<ParamField path="verify_fresh" type="bool" default="False">
  Require fresh token
</ParamField>

<ParamField path="verify_csrf" type="Optional[bool]" default="None">
  Enable CSRF verification
</ParamField>

<ParamField path="locations" type="Optional[TokenLocations]" default="None">
  Token locations to search
</ParamField>

<ResponseField name="return" type="Callable[[Request], Awaitable[TokenPayload]]">
  A FastAPI dependency function
</ResponseField>

**Raises:** `InsufficientScopeError` when token lacks required scopes

```python
# Require single scope
@app.get("/users")
def list_users(
    payload: TokenPayload = Depends(auth.scopes_required("users:read"))
):
    return {"users": []}

# Require multiple scopes (AND)
@app.delete("/users/{id}")
def delete_user(
    id: int,
    payload: TokenPayload = Depends(
        auth.scopes_required("users:read", "users:delete")
    )
):
    return {"deleted": id}

# Require any of the scopes (OR)
@app.get("/admin")
def admin_panel(
    payload: TokenPayload = Depends(
        auth.scopes_required("admin", "superuser", all_required=False)
    )
):
    return {"admin": True}

# Wildcard scope matching
@app.get("/admin/users")
def admin_users(
    payload: TokenPayload = Depends(auth.scopes_required("admin:*"))
):
    # Matches "admin:users", "admin:settings", etc.
    return {"users": []}
```

<Note>
  Supports hierarchical scopes with wildcard matching. A scope like `"admin:*"` matches any scope starting with `"admin:"` such as `"admin:users"` or `"admin:settings"`.
</Note>

## Middleware

### implicit_refresh_middleware()

FastAPI middleware that automatically refreshes access tokens when they're close to expiry.

<ParamField path="request" type="Request" required>
  Incoming request
</ParamField>

<ParamField path="call_next" type="Callable" required>
  Next middleware/endpoint in the chain
</ParamField>

<ResponseField name="return" type="Response">
  Response with refreshed token cookie if applicable
</ResponseField>

```python
from fastapi import FastAPI

app = FastAPI()

@app.middleware("http")
async def auth_middleware(request: Request, call_next):
    return await auth.implicit_refresh_middleware(request, call_next)
```

<Note>
  - Only works with cookie-based authentication
  - Refreshed tokens are not marked as fresh
  - Automatically refreshes when token expiry is within `JWT_IMPLICIT_REFRESH_DELTATIME`
  - Can be controlled using `JWT_IMPLICIT_REFRESH_ROUTE_INCLUDE/EXCLUDE` and `JWT_IMPLICIT_REFRESH_METHOD_INCLUDE/EXCLUDE`
</Note>

## Helper methods

### get_dependency()

Get an AuthXDependency instance for the current request/response context.

<ParamField path="request" type="Request" required>
  FastAPI Request object
</ParamField>

<ParamField path="response" type="Response" required>
  FastAPI Response object
</ParamField>

<ResponseField name="return" type="AuthXDependency[Any]">
  The contextual AuthX dependency object
</ResponseField>

```python
@app.get("/example")
def example(request: Request, response: Response):
    bundle = auth.get_dependency(request, response)
    token = bundle.create_access_token(uid="user123")
    bundle.set_access_cookies(token)
    return {"status": "ok"}
```

<Note>
  It's recommended to use the `auth.DEPENDENCY` or `auth.BUNDLE` property instead of calling this method directly.
</Note>

## Complete example

```python
from datetime import timedelta
from fastapi import FastAPI, Depends, Response
from authx import AuthX, AuthXConfig
from authx.schema import TokenPayload

# Configuration
config = AuthXConfig(
    JWT_SECRET_KEY="your-secret-key-here",
    JWT_ACCESS_TOKEN_EXPIRES=timedelta(minutes=15),
    JWT_REFRESH_TOKEN_EXPIRES=timedelta(days=30),
    JWT_TOKEN_LOCATION=["headers", "cookies"],
    JWT_COOKIE_CSRF_PROTECT=True
)

# Initialize AuthX
auth = AuthX(config=config)
app = FastAPI()

# Login endpoint
@app.post("/login")
def login(response: Response, username: str, password: str):
    # Verify credentials (implement your logic)
    user_id = authenticate_user(username, password)
    
    # Create tokens
    access_token = auth.create_access_token(uid=user_id, fresh=True)
    refresh_token = auth.create_refresh_token(uid=user_id)
    
    # Set cookies
    auth.set_access_cookies(access_token, response)
    auth.set_refresh_cookies(refresh_token, response)
    
    return {
        "access_token": access_token,
        "refresh_token": refresh_token
    }

# Protected endpoint
@app.get("/protected")
def protected_route(payload: TokenPayload = Depends(auth.ACCESS_REQUIRED)):
    return {
        "message": f"Hello user {payload.sub}",
        "scopes": payload.scopes
    }

# Scope-protected endpoint
@app.delete("/users/{user_id}")
def delete_user(
    user_id: str,
    payload: TokenPayload = Depends(
        auth.scopes_required("users:read", "users:delete")
    )
):
    return {"deleted": user_id}

# Refresh endpoint
@app.post("/refresh")
def refresh_tokens(
    response: Response,
    payload: TokenPayload = Depends(auth.REFRESH_REQUIRED)
):
    new_access_token = auth.create_access_token(uid=payload.sub)
    auth.set_access_cookies(new_access_token, response)
    return {"access_token": new_access_token}

# Logout endpoint
@app.post("/logout")
def logout(response: Response):
    auth.unset_cookies(response)
    return {"message": "Logged out successfully"}
```
