---
title: AuthXConfig
description: Configuration class for customizing AuthX behavior
---

The `AuthXConfig` class extends Pydantic's `BaseSettings` and provides comprehensive configuration options for JWT authentication, including token lifetimes, cookie settings, CSRF protection, and algorithm selection.

## Constructor

`AuthXConfig` inherits from Pydantic's `BaseSettings`, so it can be configured through environment variables or direct instantiation.

```python
from datetime import timedelta
from authx import AuthXConfig

# Direct instantiation
config = AuthXConfig(
    JWT_SECRET_KEY="your-secret-key",
    JWT_ACCESS_TOKEN_EXPIRES=timedelta(minutes=30),
    JWT_ALGORITHM="HS256"
)

# Or use environment variables
# JWT_SECRET_KEY=your-secret-key
# JWT_ACCESS_TOKEN_EXPIRES=1800  # seconds
config = AuthXConfig()
```

## Token configuration

### JWT_SECRET_KEY

<ParamField path="JWT_SECRET_KEY" type="Optional[str]" default="None">
  Secret key for symmetric algorithms (HS256, HS384, HS512). Required when using symmetric encryption.
</ParamField>

```python
config = AuthXConfig(JWT_SECRET_KEY="super-secret-key-change-in-production")
```

<Warning>
  Never commit your secret key to version control. Use environment variables or secure secret management.
</Warning>

### JWT_ALGORITHM

<ParamField path="JWT_ALGORITHM" type="AlgorithmType" default="'HS256'">
  Algorithm for signing JWTs. Supported values:
  - Symmetric: `HS256`, `HS384`, `HS512`
  - Asymmetric: `RS256`, `RS384`, `RS512`, `ES256`, `ES384`, `ES512`, `PS256`, `PS384`, `PS512`
</ParamField>

```python
# Symmetric (HMAC)
config = AuthXConfig(
    JWT_ALGORITHM="HS256",
    JWT_SECRET_KEY="your-secret"
)

# Asymmetric (RSA)
config = AuthXConfig(
    JWT_ALGORITHM="RS256",
    JWT_PRIVATE_KEY=private_key_string,
    JWT_PUBLIC_KEY=public_key_string
)
```

### JWT_PRIVATE_KEY

<ParamField path="JWT_PRIVATE_KEY" type="Optional[str]" default="None">
  Private key for asymmetric algorithms (RSA, ECDSA). Required when using asymmetric encryption.
</ParamField>

```python
with open("private_key.pem") as f:
    private_key = f.read()

config = AuthXConfig(
    JWT_ALGORITHM="RS256",
    JWT_PRIVATE_KEY=private_key
)
```

### JWT_PUBLIC_KEY

<ParamField path="JWT_PUBLIC_KEY" type="Optional[str]" default="None">
  Public key for asymmetric algorithms. Required when using asymmetric encryption.
</ParamField>

### JWT_ACCESS_TOKEN_EXPIRES

<ParamField path="JWT_ACCESS_TOKEN_EXPIRES" type="Optional[timedelta]" default="timedelta(minutes=15)">
  How long access tokens remain valid. Set to `None` for tokens that never expire.
</ParamField>

```python
from datetime import timedelta

config = AuthXConfig(
    JWT_ACCESS_TOKEN_EXPIRES=timedelta(hours=1)  # 1 hour
)
```

### JWT_REFRESH_TOKEN_EXPIRES

<ParamField path="JWT_REFRESH_TOKEN_EXPIRES" type="Optional[timedelta]" default="timedelta(days=20)">
  How long refresh tokens remain valid. Set to `None` for tokens that never expire.
</ParamField>

```python
config = AuthXConfig(
    JWT_REFRESH_TOKEN_EXPIRES=timedelta(days=30)  # 30 days
)
```

### JWT_TOKEN_LOCATION

<ParamField path="JWT_TOKEN_LOCATION" type="TokenLocations" default="['headers']">
  Where to look for JWTs in requests. Can be a list containing: `"headers"`, `"cookies"`, `"json"`, `"query"`
</ParamField>

```python
# Only check headers
config = AuthXConfig(JWT_TOKEN_LOCATION=["headers"])

# Check both headers and cookies
config = AuthXConfig(JWT_TOKEN_LOCATION=["headers", "cookies"])

# Multiple locations (checked in order)
config = AuthXConfig(JWT_TOKEN_LOCATION=["headers", "cookies", "json", "query"])
```

## JWT claims

### JWT_ENCODE_ISSUER

<ParamField path="JWT_ENCODE_ISSUER" type="Optional[str]" default="None">
  The issuer (`iss`) claim to include in generated tokens.
</ParamField>

```python
config = AuthXConfig(JWT_ENCODE_ISSUER="https://your-domain.com")
```

### JWT_ENCODE_AUDIENCE

<ParamField path="JWT_ENCODE_AUDIENCE" type="Optional[StringOrSequence]" default="None">
  The audience (`aud`) claim to include in generated tokens. Can be a string or list of strings.
</ParamField>

```python
config = AuthXConfig(
    JWT_ENCODE_AUDIENCE="https://api.your-domain.com"
)

# Multiple audiences
config = AuthXConfig(
    JWT_ENCODE_AUDIENCE=["api.domain.com", "app.domain.com"]
)
```

### JWT_DECODE_ISSUER

<ParamField path="JWT_DECODE_ISSUER" type="Optional[str]" default="None">
  Expected issuer when decoding tokens. If set, tokens with a different issuer will be rejected.
</ParamField>

### JWT_DECODE_AUDIENCE

<ParamField path="JWT_DECODE_AUDIENCE" type="Optional[StringOrSequence]" default="None">
  Expected audience when decoding tokens. If set, tokens without a matching audience will be rejected.
</ParamField>

### JWT_DECODE_LEEWAY

<ParamField path="JWT_DECODE_LEEWAY" type="Optional[int]" default="0">
  Leeway time in seconds for token expiration validation. Useful for clock skew between servers.
</ParamField>

```python
config = AuthXConfig(
    JWT_DECODE_LEEWAY=10  # Allow 10 seconds of clock skew
)
```

### JWT_ENCODE_NBF

<ParamField path="JWT_ENCODE_NBF" type="bool" default="True">
  Include the "not before" (`nbf`) claim in tokens.
</ParamField>

### JWT_IDENTITY_CLAIM

<ParamField path="JWT_IDENTITY_CLAIM" type="str" default="'sub'">
  The claim name used for the user identity. Standard is `"sub"` (subject).
</ParamField>

## Header configuration

### JWT_HEADER_NAME

<ParamField path="JWT_HEADER_NAME" type="str" default="'Authorization'">
  HTTP header name to read tokens from.
</ParamField>

```python
config = AuthXConfig(JWT_HEADER_NAME="X-API-Token")
```

### JWT_HEADER_TYPE

<ParamField path="JWT_HEADER_TYPE" type="str" default="'Bearer'">
  Expected token type prefix in the Authorization header. Set to empty string for no prefix.
</ParamField>

```python
# Standard: "Authorization: Bearer <token>"
config = AuthXConfig(JWT_HEADER_TYPE="Bearer")

# Custom: "Authorization: JWT <token>"
config = AuthXConfig(JWT_HEADER_TYPE="JWT")

# No prefix: "Authorization: <token>"
config = AuthXConfig(JWT_HEADER_TYPE="")
```

## Cookie configuration

### JWT_ACCESS_COOKIE_NAME

<ParamField path="JWT_ACCESS_COOKIE_NAME" type="str" default="'access_token_cookie'">
  Cookie name for access tokens.
</ParamField>

### JWT_REFRESH_COOKIE_NAME

<ParamField path="JWT_REFRESH_COOKIE_NAME" type="str" default="'refresh_token_cookie'">
  Cookie name for refresh tokens.
</ParamField>

### JWT_COOKIE_DOMAIN

<ParamField path="JWT_COOKIE_DOMAIN" type="Optional[str]" default="None">
  Domain for cookies. If None, cookies are valid for the current domain only.
</ParamField>

```python
config = AuthXConfig(
    JWT_COOKIE_DOMAIN=".your-domain.com"  # Valid for all subdomains
)
```

### JWT_COOKIE_SAMESITE

<ParamField path="JWT_COOKIE_SAMESITE" type="Optional[SameSitePolicy]" default="'lax'">
  SameSite policy for cookies: `"lax"`, `"strict"`, or `"none"`
</ParamField>

```python
config = AuthXConfig(JWT_COOKIE_SAMESITE="strict")  # Strictest CSRF protection
```

### JWT_COOKIE_SECURE

<ParamField path="JWT_COOKIE_SECURE" type="bool" default="True">
  Require HTTPS for cookies. Should be `True` in production.
</ParamField>

<Warning>
  Set to `False` only for local development. Always use `True` in production.
</Warning>

### JWT_COOKIE_HTTP_ONLY

<ParamField path="JWT_COOKIE_HTTP_ONLY" type="bool" default="True">
  Make cookies inaccessible to JavaScript (HttpOnly flag). Recommended for security.
</ParamField>

### JWT_COOKIE_MAX_AGE

<ParamField path="JWT_COOKIE_MAX_AGE" type="Optional[int]" default="None">
  Maximum age of cookies in seconds. If None, cookies expire when the browser closes (session cookies).
</ParamField>

```python
config = AuthXConfig(
    JWT_COOKIE_MAX_AGE=3600  # 1 hour
)
```

### JWT_ACCESS_COOKIE_PATH

<ParamField path="JWT_ACCESS_COOKIE_PATH" type="str" default="'/'">
  Path for access token cookies.
</ParamField>

### JWT_REFRESH_COOKIE_PATH

<ParamField path="JWT_REFRESH_COOKIE_PATH" type="str" default="'/'">
  Path for refresh token cookies.
</ParamField>

### JWT_SESSION_COOKIE

<ParamField path="JWT_SESSION_COOKIE" type="bool" default="True">
  If True, cookies expire when the browser closes (session cookies).
</ParamField>

## CSRF protection

### JWT_COOKIE_CSRF_PROTECT

<ParamField path="JWT_COOKIE_CSRF_PROTECT" type="bool" default="True">
  Enable CSRF protection for cookie-based authentication.
</ParamField>

```python
config = AuthXConfig(
    JWT_COOKIE_CSRF_PROTECT=True,
    JWT_TOKEN_LOCATION=["cookies"]
)
```

<Note>
  CSRF protection is only applied to cookie-based tokens, not header-based tokens.
</Note>

### JWT_CSRF_METHODS

<ParamField path="JWT_CSRF_METHODS" type="HTTPMethods" default="['POST', 'PUT', 'PATCH', 'DELETE']">
  HTTP methods that require CSRF validation.
</ParamField>

```python
config = AuthXConfig(
    JWT_CSRF_METHODS=["POST", "PUT", "DELETE"]
)
```

### JWT_CSRF_IN_COOKIES

<ParamField path="JWT_CSRF_IN_COOKIES" type="bool" default="True">
  Store CSRF tokens in separate cookies (accessible to JavaScript).
</ParamField>

### JWT_ACCESS_CSRF_HEADER_NAME

<ParamField path="JWT_ACCESS_CSRF_HEADER_NAME" type="str" default="'X-CSRF-TOKEN'">
  Header name for CSRF tokens in requests.
</ParamField>

### JWT_ACCESS_CSRF_COOKIE_NAME

<ParamField path="JWT_ACCESS_CSRF_COOKIE_NAME" type="str" default="'csrf_access_token'">
  Cookie name for access token CSRF.
</ParamField>

### JWT_REFRESH_CSRF_COOKIE_NAME

<ParamField path="JWT_REFRESH_CSRF_COOKIE_NAME" type="str" default="'csrf_refresh_token'">
  Cookie name for refresh token CSRF.
</ParamField>

### JWT_ACCESS_CSRF_COOKIE_PATH

<ParamField path="JWT_ACCESS_CSRF_COOKIE_PATH" type="str" default="'/'">
  Path for access CSRF cookies.
</ParamField>

### JWT_REFRESH_CSRF_COOKIE_PATH

<ParamField path="JWT_REFRESH_CSRF_COOKIE_PATH" type="str" default="'/'">
  Path for refresh CSRF cookies.
</ParamField>

### JWT_ACCESS_CSRF_FIELD_NAME

<ParamField path="JWT_ACCESS_CSRF_FIELD_NAME" type="str" default="'csrf_token'">
  Form field name for CSRF tokens when using `JWT_CSRF_CHECK_FORM`.
</ParamField>

### JWT_REFRESH_CSRF_FIELD_NAME

<ParamField path="JWT_REFRESH_CSRF_FIELD_NAME" type="str" default="'csrf_token'">
  Form field name for refresh CSRF tokens.
</ParamField>

### JWT_REFRESH_CSRF_HEADER_NAME

<ParamField path="JWT_REFRESH_CSRF_HEADER_NAME" type="str" default="'X-CSRF-TOKEN'">
  Header name for refresh CSRF tokens.
</ParamField>

### JWT_CSRF_CHECK_FORM

<ParamField path="JWT_CSRF_CHECK_FORM" type="bool" default="False">
  Check for CSRF tokens in form data in addition to headers.
</ParamField>

## Query and JSON configuration

### JWT_QUERY_STRING_NAME

<ParamField path="JWT_QUERY_STRING_NAME" type="str" default="'token'">
  Query parameter name for tokens (when using `"query"` location).
</ParamField>

```python
config = AuthXConfig(
    JWT_TOKEN_LOCATION=["query"],
    JWT_QUERY_STRING_NAME="jwt"
)
# Tokens can be passed as: /api/endpoint?jwt=<token>
```

### JWT_JSON_KEY

<ParamField path="JWT_JSON_KEY" type="str" default="'access_token'">
  JSON field name for access tokens (when using `"json"` location).
</ParamField>

### JWT_REFRESH_JSON_KEY

<ParamField path="JWT_REFRESH_JSON_KEY" type="str" default="'refresh_token'">
  JSON field name for refresh tokens (when using `"json"` location).
</ParamField>

```python
config = AuthXConfig(
    JWT_TOKEN_LOCATION=["json"],
    JWT_JSON_KEY="token"
)
# POST body: {"token": "<jwt>"}
```

## Implicit refresh configuration

<Note>
  Implicit refresh automatically creates new access tokens when they're close to expiry. Only works with cookie-based authentication.
</Note>

### JWT_IMPLICIT_REFRESH_DELTATIME

<ParamField path="JWT_IMPLICIT_REFRESH_DELTATIME" type="timedelta" default="timedelta(minutes=10)">
  Refresh tokens when they have less than this time remaining before expiry.
</ParamField>

```python
config = AuthXConfig(
    JWT_IMPLICIT_REFRESH_DELTATIME=timedelta(minutes=5)
)
```

### JWT_IMPLICIT_REFRESH_ROUTE_INCLUDE

<ParamField path="JWT_IMPLICIT_REFRESH_ROUTE_INCLUDE" type="list[str]" default="[]">
  Specific routes to enable implicit refresh. Empty means all routes (except excluded).
</ParamField>

```python
config = AuthXConfig(
    JWT_IMPLICIT_REFRESH_ROUTE_INCLUDE=["/api/users", "/api/posts"]
)
```

### JWT_IMPLICIT_REFRESH_ROUTE_EXCLUDE

<ParamField path="JWT_IMPLICIT_REFRESH_ROUTE_EXCLUDE" type="list[str]" default="[]">
  Routes to exclude from implicit refresh.
</ParamField>

```python
config = AuthXConfig(
    JWT_IMPLICIT_REFRESH_ROUTE_EXCLUDE=["/auth/login", "/auth/logout"]
)
```

### JWT_IMPLICIT_REFRESH_METHOD_INCLUDE

<ParamField path="JWT_IMPLICIT_REFRESH_METHOD_INCLUDE" type="HTTPMethods" default="[]">
  HTTP methods to enable implicit refresh. Empty means all methods (except excluded).
</ParamField>

### JWT_IMPLICIT_REFRESH_METHOD_EXCLUDE

<ParamField path="JWT_IMPLICIT_REFRESH_METHOD_EXCLUDE" type="HTTPMethods" default="[]">
  HTTP methods to exclude from implicit refresh.
</ParamField>

```python
config = AuthXConfig(
    JWT_IMPLICIT_REFRESH_METHOD_EXCLUDE=["POST", "DELETE"]
)
```

## Error configuration

### JWT_ERROR_MESSAGE_KEY

<ParamField path="JWT_ERROR_MESSAGE_KEY" type="str" default="'msg'">
  JSON key for error messages in exception responses.
</ParamField>

```python
config = AuthXConfig(JWT_ERROR_MESSAGE_KEY="error")
```

## Properties

### is_algo_symmetric

Check if the configured algorithm is symmetric (HMAC-based).

<ResponseField name="return" type="bool">
  True if algorithm is HS256, HS384, or HS512
</ResponseField>

```python
if config.is_algo_symmetric:
    print("Using symmetric encryption")
```

### is_algo_asymmetric

Check if the configured algorithm is asymmetric (RSA/ECDSA-based).

<ResponseField name="return" type="bool">
  True if algorithm is RS*, ES*, or PS*
</ResponseField>

```python
if config.is_algo_asymmetric:
    print("Using asymmetric encryption")
```

### private_key

Get the private key for token encoding.

<ResponseField name="return" type="str">
  The private key (JWT_PRIVATE_KEY for asymmetric, JWT_SECRET_KEY for symmetric)
</ResponseField>

**Raises:** `BadConfigurationError` if key is not set or algorithm is unsupported

### public_key

Get the public key for token decoding.

<ResponseField name="return" type="str">
  The public key (JWT_PUBLIC_KEY for asymmetric, JWT_SECRET_KEY for symmetric)
</ResponseField>

**Raises:** `BadConfigurationError` if key is not set or algorithm is unsupported

## Methods

### has_location()

Check if a specific token location is enabled.

<ParamField path="location" type="str" required>
  Location to check: `"headers"`, `"cookies"`, `"json"`, or `"query"`
</ParamField>

<ResponseField name="return" type="bool">
  True if the location is in JWT_TOKEN_LOCATION
</ResponseField>

```python
if config.has_location("cookies"):
    print("Cookie authentication is enabled")
```

## Complete example

```python
from datetime import timedelta
from authx import AuthXConfig

config = AuthXConfig(
    # Token settings
    JWT_SECRET_KEY="your-secret-key-here-change-in-production",
    JWT_ALGORITHM="HS256",
    JWT_ACCESS_TOKEN_EXPIRES=timedelta(minutes=15),
    JWT_REFRESH_TOKEN_EXPIRES=timedelta(days=30),
    
    # Token locations
    JWT_TOKEN_LOCATION=["headers", "cookies"],
    
    # JWT claims
    JWT_ENCODE_ISSUER="https://your-domain.com",
    JWT_ENCODE_AUDIENCE="https://api.your-domain.com",
    JWT_DECODE_ISSUER="https://your-domain.com",
    JWT_DECODE_AUDIENCE="https://api.your-domain.com",
    JWT_DECODE_LEEWAY=10,
    
    # Cookie settings
    JWT_COOKIE_SECURE=True,
    JWT_COOKIE_SAMESITE="lax",
    JWT_COOKIE_DOMAIN=".your-domain.com",
    JWT_COOKIE_MAX_AGE=86400,  # 24 hours
    
    # CSRF protection
    JWT_COOKIE_CSRF_PROTECT=True,
    JWT_CSRF_METHODS=["POST", "PUT", "PATCH", "DELETE"],
    JWT_CSRF_IN_COOKIES=True,
    
    # Implicit refresh
    JWT_IMPLICIT_REFRESH_DELTATIME=timedelta(minutes=5),
    JWT_IMPLICIT_REFRESH_ROUTE_EXCLUDE=["/auth/login", "/auth/logout"],
)
```
